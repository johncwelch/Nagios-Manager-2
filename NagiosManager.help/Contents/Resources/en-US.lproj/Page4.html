User Manager

The User Manager tab is where you manage users on the selected Nagios server. You can:

  • Add users

  • Delete users

  • Get information about a user: full name, username, user id, email address, enabled status (1 or 0)

Please note that due to limitations in the REST API, you can’t update or change a user’s settings from Nagios Manager. 


User Manager Tab

As with all tabs, the User Manager Tab has the same three basic  functional areas . The  main table  contains the User Name (Full Name) and User ID for each user on a given Nagios server. It does not show what level user they are. (i.e. admin, read-only, other.) When you select a user in the  main table , the  status display  will show the following info for that user:

  • Full Name

  • Username (aka login name)

  • User ID

  • Email Address

  • User Enabled status, (1 for enabled, 0 for disabled.)


Selecting a Server

The “Select Server” popup is where you select the specific server whose users you are managing. If there are multiple servers in  Nagios Manager’s preferences file , then the first server is decided by its order in the preferences file. The first user returned for a given server is selected by default in the  main table . If you change servers, the  main table  will be reloaded with the users for the server you select.


Selecting and Deleting a User

To select a user, click on that user’s row in the  main table . To delete, either double-click on a row in the  main table , or select the user you want to delete and click the “Delete Selected User” button. You’ll be asked if you’re really sure you want to delete the user. If you click “OK”, the user will be deleted.

Adding a User

When adding a user, the Username, Password, Full Name and Email address are required. Please do not put spaces or special characters in the username, that makes Nagios sad. When adding a user, there are certain settings that are not shown, and cannot be changed within Nagios Manager:

  • Default Language is set to “xi default” or the default language for the specified Nagios server

  • Date format is set to the default date format for the specified Nagios server

  • Number format is set to the default number format for the specified Nagios server

  • Force Password Change upon login is always enabled

  • Email account info to the user you’re creating is always enabled

  • Make this user a monitoring contact is always enabled

  • Enable notifications for this user is always enabled

  • You cannot enable API access from Nagios Manager. This is a deliberate design decision.


Auth Type  - This is for use with a Nagios server that was set up in Nagios Manager as having an AD or LDAP external authentication server. If the Nagios server you’re working against was not set up with an AD/LDAP auth server, then this will be locked to “Local” and disable, as that’s the only functional option in that case. If the server has an external auth server function, then you select which one of three options to use here: Local, AD, LDAP. Please make sure the one you pick matches the server setup for the Nagios server you’re working against. If you pick “Local”, that’s all you get. If you pick AD or LDAP, you can have a local auth option, as seen below. 

IMPORTANT : If you are using AD or LDAP, then the Username: field  must  match the AD or LDAP username for AD/LDAP auth to work. 


Allows Local Auth  - This option allows you to create a local password for an account that would normally use AD or LDAP authentication. If the connection to the external auth server is unreliable, this is a good option. It is disabled by default. Note that if this is not enabled, and you choose AD or LDAP as the Auth Type, the Password: field is disabled, since if you’re only using AD or LDAP auth for a user, there is no need to enter a local password. If the Auth Type is set to Local, this control is disabled and unchecked.


There are three levels of Nagios user: Admin, Read Only, User. The default setting on application launch is User, with permissions that will allow that user to interact with Nagios in a useful manner sans being an admin.

  • Admin  - All permissions enabled save API access, that must be enabled within Nagios itself. Selecting the “Admin” radio button locks out Read Only as an option. 

  • Read Only  - Only available if the “User” radio button is selected. Removes all other permissions. Note: As it removes “Can See All Objects”, by default, this user will not be able to actually see any Nagios host or service on the selected server. That will have to be enabled within Nagios itself.

  • User  - The default user level. The default permissions for this are: Can See All Objects:, Can Control All Objects:, Can Access Advanced Features:. For detailed explanation(s) of these permission levels, use the online help within Nagios XI. 

Once you have set up the user you want to add, click the “Add User” button, and the add user REST command will be submitted to Nagios, and the user list automatically reloaded. If you do not see the user you just added, wait a few minutes. Sometimes, Nagios is slow. Reselect the server in the “Select Server:” popup to force a user list reload. 

If you get an error in the  status display , you either left out a necessary piece of information, or an error with Nagios occurred. For the former, double-check what you entered in terms of user name/password/etc. For the latter, refer to the error message and Nagios’ own logs.
