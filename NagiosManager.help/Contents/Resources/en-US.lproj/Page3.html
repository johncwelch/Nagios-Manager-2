Server Manager

The Server manager Tab is where you manage Nagios servers. These are the actual Nagios servers themselves, as opposed to say, servers you may be monitoring via Nagios. Within the server Tab, you can:

  • Add a Nagios server

  • Delete a single Nagios Server

  • Delete All Servers 

  • Reload Servers (re-read the on-disk preferences file)

Nagios Manager keeps its preferences in ~/Library/Preferences/bynkii.com.Nagios-Manager.plist. Currently this file is not encrypted at all, so please be careful with it. (I’m looking at adding that at some point.)


Server Manager Tab


The server tab  main table  holds a list of Nagios servers, one server per row. Each row has:

  • The Server Name

  • The Server URL

  • The Server API Key

When you select a server in the  main table , the  status display  will show some basic information for the server you’ve selected. On application launch, the  status display  will automatically show status for the first server in the list. 


Selecting and Deleting a Server

Selecting a server in the  main table  also allows you to delete it from Nagios Manager. You can delete the server by selecting it, then clicking the “Delete Selected Server(s) button, or by double-clicking the server you want to delete. Either way will display a “Are you sure you want to do this” dialog. If you click “OK” in the dialog, the server will be deleted. (Note that currently, you can’t select multiple servers. But soon, you will be able to do that.)


Adding a Server

To add a server, at a minimum you have to have the server name, the server URL, the server API key and how the server will authenticate:

Basic Settings

  • The name of the server goes in the  Server Name  field. This can be anything you want it to be, it is only there for your ease of identification. This is also the name you will use when selecting servers in the other tabs, so you should make it reasonably short and immediately meaningful. “Network Nagios Server” is immediately meaningful, “NNS34QPK5” less so.

  • The URL for the server goes in the  Server URL  field.  IMPORTANT :  ONLY  enter in the main DNS name or IP address for the server, i.e. http(s)://my.nagios.com/ or https://10.10.10.10/. Do not enter anything else, as at the moment, the basic URL form is grafted on the end of what you enter here. If you enter the wrong thing here, Nagios Manager will not be able to properly communicate with your server. 

  • Server API Key : This is available from your user account on the Nagios Server you’re adding to Nagios Manager. For best results, it should be an admin account API Key. There’s not a really good way for Nagios Manager to validate what a given key can or cannot do. If you enter an API key that is not an admin key, your results will be…uncertain.

  • Uses AD/LDAP Auth  - Fairly simple. If you want to use an external authentication server, enable this checkbox. if you want to use local  only , leave this unchecked. Note that if you have a combination of local/external users, enable this. You don’t have to use external auth, but if you don’t set it

Authentication Server Settings


  • Common Authentication Server Controls

  • Connection Method - Choose either AD or LDAP. You get one external option per server, AD or LDAP. Not both. If there’s actually a real need for both, I can change that later.

  • Security Level - Options are None, TLS, SSL. The value here should always match the value you have set up within the AD/LDAP setup on the specific Nagios server. At this point, you should probably not be using SSL.

  • Base DN: the base DN of your AD or LDAP tree, for example, “DC=nagios, DC=com”. Each DC= entry must be separated by a comma

  • Active Directory Settings

  • AD Domain Suffix - this is primarily used for Active Directory, it’s the “new” (Since 2000) “Kerberos” style of domain/user name. So instead of “NAGIOS\john”, it’d be “john@nagios.com”. 

  • AD Domain Controllers - A comma-delimited list of DNS names and/or IP addresses for your AD/LDAP domain controllers, (DCs). AD setups can have many, many DCs, 

  • LDAP Settings

  • LDAP Host - The IP address or DNS name of the LDAP server you want to use for authentication. Currently, you get one, choose wisely.

  • LDAP Port - The numerical port you use to connect to the LDAP Host on. Generically, this is 389 for unsecure connections and 636 for TLS/SSL


Add Server  - Once you’ve filled in the information for the server, this adds the server to the Nagios Manager prefs. IMPORTANT: this only adds it to the application prefs. It does not in any way, shape, or form configure a Nagios server. If what you have here does not match the server you’re pointing Nagios Manager at, you’re not going to be able to use Nagios Manager with that Nagios server.


Reload Servers

This is simply a way to reload the preferences file data into the app. It’s not something you’ll really need to do much, unless you’re directly editing the file outside of Nagios Manager. That’s maybe not the best idea, but if you have a lot of Nagios Servers, doing them one at a time can be tedious, so this exists primarily for that use case. If you want to twiddle the prefs file directly, I’d recommend adding one server via the Nagios Manager UI so that you can get an idea of the formatting conventions for the plist.


Delete All Servers

As it implies, this wipes all the servers out of the preferences file. It does not delete the preference file, just removes the server data from it.
